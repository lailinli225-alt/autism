# 自闭症家长辅导产品 UI 设计规范

参考方向：站酷案例《宠物派APP改版 | UI设计》的移动端产品化表达、模块化展示、服务闭环组织方式。不得复制其视觉资产、配色、插画或版式细节；本规范只吸收“清晰分区、轻量卡片、完整服务路径、友好但有秩序”的设计方法。

## 1. 产品气质

### 关键词

- 温和
- 稳定
- 专业
- 可执行
- 有希望但不兴奋

### 不做什么

- 不做潮玩感、强刺激、强对比、强装饰。
- 不使用大面积高饱和渐变。
- 不用夸张表情包、过度可爱化 IP、强营销式话术。
- 不把家长的问题包装成“打卡挑战”或“成长奇迹”。

### 情绪目标

家长打开页面时，应该感觉：

- “我可以慢慢说清楚。”
- “系统不会评判我。”
- “我能得到一个今天能做的小步骤。”
- “孩子的变化可以被记录和看见。”

| 状态 | 色值 | 使用方式 |
| --- | --- | --- |
| 成功 | `#3F873F` | 完成、保存、有效信号 |
| 警示 | `#B65F2B` | 需要注意但不紧急 |
| 风险 | `#B64A3A` | 自伤、攻击、走失等安全风险 |
| 待处理 | `#8A918B` | 未填写、未生成 |

### 色彩原则

- 背景必须低饱和，优先使用绿灰、米白、浅暖黄。
- 主色只用于关键动作，不要让页面满屏绿色。
- 风险色只能用于真正安全风险，避免制造焦虑。
- 每屏最多 1 个主按钮，最多 2 个强调色。

## 3. 字体与排版

### 字体

- 中文：`PingFang SC` / `Noto Sans SC` / `Microsoft YaHei`
- 英文与数字：`Inter`

### 字号

| 层级 | 字号 | 字重 | 用途 |
| --- | --- | --- | --- |
| 页面标题 | 20-22 | 760 | 页面主标题 |
| 模块标题 | 16-18 | 740 | 卡片标题、分区标题 |
| 正文 | 14-15 | 400-560 | 说明、建议 |
| 标签 | 12-13 | 650-720 | 状态、分类 |
| 辅助说明 | 11-12 | 400-560 | 输入提示、时间、说明 |

### 排版原则

- 行高保持在 `1.5-1.7`，让长文本可读。
- 不使用负字距。
- 不用超大标题制造情绪压迫。
- 医疗、干预建议类文本必须分段，不堆成长段落。

## 4. 圆角、阴影与间距

### 圆角

| 元素 | 圆角 |
| --- | --- |
| 页面卡片 | 12px |
| 功能卡片 | 10-12px |
| 输入框 | 8-10px |
| 标签/胶囊 | 999px |
| 主按钮 | 12px 或 24px 胶囊 |

### 阴影

默认少用阴影，主要依靠边框和浅背景区分层级。

```css
--shadow-soft: 0 8px 24px rgba(42, 73, 45, 0.08);
--shadow-action: 0 8px 18px rgba(63, 135, 63, 0.18);
```

### 间距

| 场景 | 间距 |
| --- | --- |
| 页面左右边距 | 20px |
| 卡片内边距 | 16px |
| 模块上下间距 | 18-24px |
| 表单项间距 | 14-16px |
| 标签间距 | 8px |

## 5. 组件规范

### 主按钮

- 文案必须是明确动作：`保存并生成复盘`、`生成干预建议`、`记录今天情况`。
- 不使用空泛文案：`开始探索`、`立即开启奇迹`。
- 主按钮颜色：`care-green-600`。
- 禁用态透明度 60%，保留文字可读。

### 输入框

用于家长描述孩子行为，应尽量降低书写压力。

要求：

- 提供示例，但示例不要太极端。
- 支持语音输入入口。
- 输入提示用“你可以这样写”，不要用“请准确填写”。
- 长文本输入框高度不低于 112px。

示例提示：

```text
例如：今天在商场排队时，孩子听到广播后捂耳朵，尖叫了大约 2 分钟。后来我带他到安静角，3 分钟后平静。
```

### 卡片

卡片类型：

- 今日目标卡
- 干预记录卡
- AI 结构化整理卡
- ABC 记录卡
- 趋势复盘卡
- 安全提醒卡

卡片原则：

- 每张卡只承载一个任务。
- 标题短，正文具体。
- 不在卡片里再套卡片。
- AI 输出必须结构化，不直接甩一整段长答案。

### 标签

标签用于快速识别，不用于装饰。

推荐标签：

- `等待练习`
- `情绪调节`
- `沟通表达`
- `感官压力`
- `已完成`
- `需复盘`
- `安全优先`

### 图标

使用线性图标，圆角端点，视觉重量轻。

推荐图标语义：

- 家庭：双人/手心
- 记录：笔记/清单
- AI：闪光/整理
- 趋势：折线
- 安全：盾牌
- 情绪：温度计
- 感官：耳朵/波纹

图标不要使用卡通化医疗符号，不使用夸张脑部、谜图片等容易标签化孩子的视觉元素。

## 6. 页面模板

### 首页

目标：让家长快速进入今天最需要的任务。

结构：

1. 顶部问候：孩子昵称、今日状态。
2. 今日建议：只给 1 个优先事项。
3. 快捷入口：记录干预、问 AI、查看趋势、给老师生成说明。
4. 最近变化：成功次数、情绪恢复时间、常见触发点。

### 干预记录页

目标：家长每天 3 分钟完成记录。

结构：

1. 日期和孩子。
2. 今日目标。
3. 自然语言记录。
4. 快速评分：情绪强度、辅助程度、配合度。
5. AI 结构化整理：触发点、行为、后果、干预方式、有效信号、下次调整。
6. 保存并生成复盘。

### AI 分析页

目标：把家长的问题转成具体方案。

结构：

1. 当前问题摘要。
2. 可能原因。
3. 立即处理。
4. 7 天练习。
5. 不建议做什么。
6. 何时需要专业支持。

### 趋势页

目标：让家长看到可追溯变化。

结构：

1. 本周变化摘要。
2. 关键指标：频率、持续时间、恢复时间、辅助等级。
3. 触发点排行。
4. 有效方法排行。
5. 下周建议。

### 孩子档案页

目标：逐步完善，不在注册时制造负担。

结构：

1. 基础信息。
2. 语言能力。
3. 感官特点。
4. 当前干预。
5. 安全风险。
6. 学校/家庭场景。

## 7. 文案语气

### 语气原则

- 不评判。
- 不保证疗效。
- 不制造内疚。
- 不用过度鸡汤。
- 把建议写成小步骤。

### 推荐表达

| 不推荐 | 推荐 |
| --- | --- |
| 你需要坚持训练 | 今天先只练一个小步骤 |
| 孩子不配合 | 孩子当前可能承受的要求过高 |
| 这个方法一定有效 | 可以先尝试 3 天，并记录变化 |
| 家长要多努力 | 先降低任务难度，让家长和孩子都能成功 |
| 问题行为 | 当前行为表现 |

### 安全提醒文案

```text
如果出现自伤、攻击、走失风险，或家长无法保证现场安全，请优先联系专业人员或就近求助。本工具只提供家庭支持参考，不替代诊疗。
```

## 8. AI 输出规范

AI 不直接输出大段建议，必须整理成以下结构：

```json
{
  "可能原因": [],
  "当前处理": [],
  "家庭练习": [],
  "记录指标": [],
  "下次调整": [],
  "安全提醒": ""
}
```

干预记录页 AI 结构：

```json
{
  "触发点": "",
  "孩子行为": "",
  "家长干预": "",
  "孩子反应": "",
  "有效信号": "",
  "下次调整": "",
  "可追踪指标": []
}
```

## 9. 可访问性与情绪安全

- 文字与背景对比度不低于 4.5:1。
- 不使用快速闪烁、跳动动画。
- 错误提示不使用红色大面积背景。
- 所有提交动作都要有保存状态。
- AI 生成等待时间必须有明确反馈。
- 家长输入内容失败时不能丢失。

## 10. 视觉落地参考

### 推荐风格

- 浅绿灰背景。
- 白色或暖白卡片。
- 少量暖黄提示。
- 线性图标。
- 数据以小组件呈现。
- 关键建议用清晰分组，而不是大段聊天气泡。

### 不推荐风格

- 医疗蓝白冷冰冰。
- 儿童乐园式高饱和。
- 大面积紫蓝渐变。
- 过多拟物插画。
- 满屏 AI 聊天气泡。
- 强烈 IP 人设或夸张表情。

## 11. 一句话设计准则

每一个页面都应该帮助家长完成一个动作：说清楚、看明白、做一步、记下来、能复盘。
